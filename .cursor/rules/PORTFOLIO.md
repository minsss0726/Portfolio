# 포트폴리오 사이트 개발 룰

## 프로젝트 개요

### 주제

Next.js 기반 개발자 포트폴리오 웹사이트 개발

### 목표

- 개발자의 경력, 프로젝트, 기술 스택을 효과적으로 소개하는 포트폴리오 사이트 구축
- 인터랙티브하고 현대적인 UI/UX 제공
- 반응형 디자인으로 다양한 디바이스에서 최적화된 경험 제공
- SEO 최적화를 통한 검색 엔진 노출 향상

## 개발 방식

### 기술 스택

- **프레임워크**: Next.js (App Router 사용)
- **언어**: TypeScript
- **스타일링**: Tailwind CSS 또는 styled-components
- **상태 관리**: 필요시 Zustand 또는 React Context
- **데이터 페칭**: Next.js 내장 fetch 또는 TanStack Query
- **애니메이션**: Framer Motion 또는 CSS transitions
- **배포**: Vercel (권장) 또는 기타 호스팅 플랫폼

### 필수 섹션

1. **홈/히어로 섹션**
   - 인터랙티브한 텍스트 애니메이션
   - 간단한 자기소개
   - 주요 기술 스택 표시

2. **프로필 섹션**
   - 프로필 이미지
   - 간단한 소개
   - 이력서 다운로드 버튼
   - 소셜 링크 (GitHub, LinkedIn, Blog 등)

3. **학력 섹션**
   - 학교명, 전공, 기간
   - 로고 이미지 포함

4. **자격증/수료증 섹션**
   - 자격증 및 수료증 목록
   - 발행 기관, 발행일
   - 수료증 이미지 갤러리

5. **경력사항 섹션**
   - 회사명, 재직 기간, 직책
   - 주요 업무 내용 (상세 설명)
   - 사용 기술 스택 태그
   - 프로젝트별 상세 설명

6. **프로젝트 섹션**
   - 프로젝트 카드 형태로 표시
   - 프로젝트 이미지/썸네일
   - 프로젝트 설명
   - 사용 기술 스택
   - 링크 (GitHub, 데모 사이트 등)

7. **블로그 섹션** (선택)
   - 최근 블로그 포스트 목록
   - 외부 블로그 링크

### 디자인 원칙

- **모던하고 미니멀한 디자인**: 깔끔하고 전문적인 느낌
- **다크 모드 지원**: 사용자 선호도에 따른 테마 전환
- **인터랙티브 요소**: 호버 효과, 스크롤 애니메이션
- **타이포그래피**: 가독성 좋은 폰트 선택
- **색상**: 브랜드 컬러를 일관되게 사용

### 개발 가이드라인

#### 코드 구조

```
app/
  ├── (routes)/
  │   ├── page.tsx          # 홈 페이지
  │   ├── about/
  │   ├── projects/
  │   └── blog/
  ├── components/
  │   ├── sections/         # 섹션별 컴포넌트
  │   ├── ui/               # 재사용 가능한 UI 컴포넌트
  │   └── layout/           # 레이아웃 컴포넌트
  ├── lib/                  # 유틸리티 함수
  ├── data/                 # 정적 데이터 (경력, 프로젝트 등)
  └── styles/               # 전역 스타일
```

#### 컴포넌트 설계

- **재사용성**: 공통 컴포넌트는 `components/ui/`에 분리
- **섹션별 분리**: 각 섹션은 독립적인 컴포넌트로 구성
- **타입 안정성**: TypeScript 인터페이스로 데이터 구조 정의
- **Props 타입**: 모든 컴포넌트에 명시적 타입 정의

#### 성능 최적화

- **이미지 최적화**: Next.js Image 컴포넌트 사용
- **코드 스플리팅**: 동적 import 활용
- **메타데이터**: SEO를 위한 적절한 meta tags 설정
- **로딩 상태**: Suspense와 loading.tsx 활용

#### 접근성

- **시맨틱 HTML**: 적절한 HTML 태그 사용
- **ARIA 라벨**: 스크린 리더 지원
- **키보드 네비게이션**: 모든 인터랙티브 요소 접근 가능
- **색상 대비**: WCAG 가이드라인 준수

### 데이터 관리

- **정적 데이터**: TypeScript/JSON 파일로 관리
- **타입 정의**: 모든 데이터에 대한 인터페이스 정의
- **데이터 구조화**: 섹션별로 명확하게 분리

### 배포 및 운영

- **환경 변수**: 민감한 정보는 환경 변수로 관리
- **에러 처리**: 에러 바운더리 및 적절한 에러 페이지
- **모니터링**: 필요시 에러 추적 도구 연동
- **성능 모니터링**: Core Web Vitals 추적
